# Contributing

Keep changes small and focused.

- Match the existing firmware style and documentation structure.
- Avoid introducing new dependencies unless they are required for the fix.
- Do not commit build outputs, local machine files, or credentials.
- Update the relevant README or setup doc when behavior changes.
- Run the smallest useful build or test command for the area you touched.

If you are adding board support, document which Zephyr or Arduino target was
actually verified and which targets are only expected to work.
